# 战场恢复手册 — 卡壳时的具体打法

> "善败者不乱。" — 孙子
>
> "敌进我退，敌驻我扰，敌疲我打，敌退我追。" — 毛泽东游击战十六字诀

---

## 何时使用此手册

连续失败2次以上，且不清楚下一步该怎么做时，打开这份手册。

这不是放弃，是重新侦察、重新部署。

---

## 第一步：战场评估（3分钟内完成）

停下来。不要继续执行。先回答这三个问题：

```
问题1：我确定知道什么？（列出已验证的事实，不是猜测）
问题2：我以为知道但没验证的是什么？（潜在错误假设）
问题3：我完全不知道的是什么？（情报盲区）
```

**关键洞察**：大多数卡壳，根源在"问题2"——用了错误的假设但没有意识到。

---

## 第二步：按卡壳类型选择恢复策略

### 类型A：方向错了（打错了主要矛盾）

**症状**：改了很多东西，问题还在；或者修了一个问题，出现了新问题。

**恢复策略 — 毛泽东矛盾论**：
1. 停止所有修改
2. 重新执行五问根因法，不带任何前置假设
3. 问自己："如果我完全搞错了方向，正确的方向会是什么？"
4. 从最简单可能的原因开始验证，不是最复杂的

```
[根因重建]
症状：<重新描述你看到的现象>
最简单的解释：<奥卡姆剃刀——最少假设的解释是什么？>
验证方法：<如何用一个命令/操作验证这个解释？>
```

---

### 类型B：情报不足（没有调查就没有发言权）

**症状**：做了很多，但感觉在猜；或者找不到错误的原因。

**恢复策略 — 稻盛和夫"在现场"**：
1. 列出所有你"以为"知道的事情
2. 对每一条：我是怎么知道的？是读了原始材料，还是猜的？
3. 把"猜的"那些都用工具去验证

```
[情报核查]
假设1：<你以为成立的事情> → 验证方法：<> → 结果：✓/✗
假设2：<你以为成立的事情> → 验证方法：<> → 结果：✓/✗
...
最令人惊讶的发现：<哪个"以为"被证伪了？>
```

---

### 类型C：思路固化（在原地打转）

**症状**：尝试了很多方法，但本质上都是同一个方向的变体。

**恢复策略 — 芒格反转思维**：
1. 写下你一直在做的方向
2. 强制反转："如果这个方向完全错了，对立的方向是什么？"
3. 用对立方向重新看问题

**反转练习**：
```
一直在做：<当前方向>
反转后变成：<对立方向>
从对立方向看，问题是什么？
从对立方向，解法是什么？
```

**类比迁移（芒格第二工具）**：
```
这个问题像什么其他领域的问题？
那个领域的解法是什么？
能类比迁移到这里吗？
```

---

### 类型D：执行层卡死（知道怎么做但做不了）

**症状**：有思路，但每次执行都遇到新的障碍；工具/环境/权限/依赖有问题。

**恢复策略 — 毛泽东游击战：最小可行路径**：
1. 找到最小的、确定能成功的一步
2. 做这一步，只这一步
3. 成功后扩大一圈

```
[最小可行路径]
最终目标：<要到达的地方>
能100%成功的最小第一步：<>
验证方法：<>
第一步成功后的第二步：<>
...
```

---

### 类型E：问题定义模糊（战场地图缺失）

**症状**：不清楚"成功"是什么样子；或者问题描述很模糊，不知道从哪入手。

**恢复策略 — 孙子"庙算"**：
1. 在动手之前，先在脑子里把成功和失败都想清楚
2. 定义"什么情况下我知道我赢了"
3. 定义"什么情况下我知道我输了/走错了"

```
[成功定义]
成功的标志：<什么样的输出/结果证明问题解决了？>
失败的早期信号：<如果走错方向，最早会在哪里看到迹象？>
最小验证点：<能证明方向正确的最小测试是什么？>
```

---

## 第三步：恢复后的首选行动

恢复策略执行后，第一个行动必须是**可以立即验证**的：

- 代码场景：改一行，跑一次，看结果
- 搜索场景：搜一个关键词，读前三个结果
- 配置场景：改一个配置项，重启，看日志
- 分析场景：验证一个核心假设

**不允许的首选行动**：改十个地方、重写整个模块、一次性执行复杂操作。

---

## 第四步：恢复后的进化记录

恢复成功后，写一条进化记录：

```
[恢复记录] <日期>
卡壳类型：A/B/C/D/E
根本原因：<为什么卡住了？>
恢复策略：<用了哪个策略？>
关键转折点：<什么发现让局面转变了？>
下次预防：<类似情况下，一开始就应该做什么？>
思想家洞见：<哪位思想家的哪个原则在这里起了作用？>
```

## 快速查找表

| 感觉... | 可能是... | 用... |
|--------|---------|------|
| 改了很多，没进展 | 类型A：方向错了 | 矛盾论重建 |
| 不知道为什么出错 | 类型B：情报不足 | 在现场验证 |
| 换了参数还是不行 | 类型C：思路固化 | 反转思维 |
| 工具/环境问题 | 类型D：执行层卡死 | 最小可行路径 |
| 不知道做对了算什么 | 类型E：定义模糊 | 庙算定义成功 |
