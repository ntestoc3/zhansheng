# 打胜仗思维激活引擎

> "没有调查，就没有发言权。" — 毛泽东
>
> "知己知彼，百战不殆。" — 孙子

---

## 这是什么？先讲个故事

你有没有遇到过这种情况：

让 AI 帮你调一个 bug，它改了三次，每次都是换个参数，问题还在。或者你问它一个复杂问题，它给你一个"差不多"的答案，你总感觉哪里不对，但说不清楚。

问题出在哪？

**AI 在用蛮力，不在用脑子。**

它没有先搞清楚"这场仗该怎么打"，就直接冲上去了。改参数、换配置、重试——这是打地鼠，不是解决问题。

这个项目做的事情很简单：**给 AI 装上一套思想武器**，让它在动手之前先想清楚，在执行过程中有章法，在失败之后知道怎么调整。

思想来源：毛泽东思想、孙子兵法、稻盛和夫哲学、查理·芒格多元思维模型、王阳明知行合一。

这些不是装饰，是真正有效的认知框架，被这个 skill 翻译成了 AI 可以直接执行的工作协议。

---

## 它能做什么？

激活这个 skill 之后，AI 会：

1. **先侦察，再动手** — 不读源码就猜、不查文档就断言的情况会消失
2. **找主要矛盾** — 不打地鼠，找根本原因，一击即穿
3. **多角度分析** — 用至少三个不同思维模型交叉验证，不靠单一框架
4. **知行合一** — 说"完成了"之前必须有实际验证的证据
5. **失败后升级** — 每次失败都触发更深的侦察，而不是重复同样的错误
6. **战后复盘** — 每次任务结束沉淀方法论，越用越强

## 怎么安装？

这个 skill 是为 **Claude Code**（Anthropic 官方 CLI 工具）设计的。

### 第一步：确认你有 Claude Code

```bash
claude --version
```

如果没有，先安装：

```bash
npm install -g @anthropic-ai/claude-code
```

### 第二步：找到你的 Claude Code 配置目录

Claude Code 的 skills 和 commands 放在用户级别的配置目录里：

```bash
# macOS / Linux
~/.claude/

# 查看是否存在
ls ~/.claude/
```

### 第三步：复制文件

```bash
# 复制 skill
cp -r skills/zhansheng ~/.claude/skills/

# 复制 slash command
cp commands/zhansheng.md ~/.claude/commands/
```

### 第四步：验证安装

在任意项目目录启动 Claude Code，输入：

```
/zhansheng
```

如果 skill 加载成功，AI 会进入"打胜仗"工作模式。

## 怎么用？

### 方式一：用 slash command 触发（推荐新手）

在 Claude Code 对话中，直接输入：

```
/zhansheng
```

然后描述你的任务。AI 会自动进入战略思维模式，先做侦察，再动手。

**适合场景**：
- 遇到复杂 bug，不知道从哪入手
- 需要设计一个架构方案
- 任务路径不清晰，需要先想清楚再做

### 方式二：在任务描述里直接提

```
帮我调查这个报错，用打胜仗思维分析一下主要矛盾在哪里
```

### 方式三：卡壳时打开恢复手册

如果 AI 连续失败两次以上，让它参考 `recovery-playbook.md`：

```
你已经失败两次了，打开 recovery-playbook，先做战场评估，再继续
```

## 一个完整的使用例子

**场景**：你的 Next.js 项目部署到生产环境后，某个 API 路由返回 500，本地没问题。

**不用这个 skill 时**，AI 可能会：
- 让你检查环境变量（猜）
- 让你重新部署（猜）
- 说"可能是服务器配置问题"（猜）

**用了这个 skill 之后**，AI 会先做战场侦察：

```
[战场侦察]
任务本质：生产环境 API 500，本地正常，找出根本原因
主要矛盾：生产和本地的差异是什么？
已知情报：本地正常，说明代码逻辑本身没问题
情报盲区：生产环境的日志是什么？环境变量是否完整？依赖版本是否一致？
侦察行动：读生产日志全文 → 对比环境变量 → 检查 package.json 版本锁定
```

然后用五问根因法追到底，而不是猜。

## 核心思想速查

| 思想家 | 核心原则 | 在 AI 工作中的应用 |
|--------|---------|-----------------|
| 毛泽东 | 没有调查就没有发言权 | 先读源码/文档，再给答案 |
| 毛泽东 | 集中优势兵力，各个歼灭 | 找主要矛盾，不打分散仗 |
| 孙子 | 知己知彼，百战不殆 | 彻底理解问题本质再动手 |
| 稻盛和夫 | 在现场 | 答案在代码/日志里，不在猜测里 |
| 查理·芒格 | 多元思维模型 | 用3+个框架交叉验证答案 |
| 王阳明 | 知行合一 | 说完成了必须有验证证据 |

## 与其他 skill 搭配

这个 skill 可以和 PUA 系列 skill 叠加使用：

```
① 任务开始 → zhansheng：战场侦察 + 主矛定位
② 执行中   → zhansheng：打胜仗七步法
③ 第1次失败 → zhansheng：失败模式诊断
④ 第2次失败 → pua:high-agency Recovery Protocol
⑤ 仍然失败 → pua:pua 接管施压
⑥ 任务完成 → zhansheng：战后复盘
```

单独用 zhansheng 侧重**思想深度**，叠加 pua 系列侧重**执行纪律**。

## 常见问题

**Q：这和 PUA skill 有什么区别？**

PUA 靠外部压力驱动——"你这都解决不了？"。打胜仗思维靠认知驱动——"你拿什么打这场仗？主要矛盾找到了吗？"前者让 AI 更努力，后者让 AI 更聪明。

**Q：每次都要手动触发吗？**

可以在项目的 `CLAUDE.md` 里写入常驻指令，让 AI 在这个项目里默认使用打胜仗思维模式。

**Q：recovery-playbook.md 什么时候用？**

当 AI 连续失败两次以上，且不清楚下一步该怎么做时。手册里有五种卡壳类型的具体恢复策略，按症状对号入座。
